WHAT IS DECAMP?
---------------
Decamp is a comprehensive AI-powered personal assistant web application that
integrates directly with the user's Gmail and Google Calendar. It leverages
Google's Gemini AI model to:

  1. Retrieve and analyze emails from the user's Gmail inbox
  2. Prioritize emails into three urgency tiers:
       - IMMEDIATE ATTENTION (critical, time-sensitive)
       - NEEDS RESPONSE (important but not urgent)
       - CAN WAIT (newsletters, low-priority)
  3. Summarize long email threads into concise bullet points
  4. Group emails by category (work, personal, finance, meetings, forms, etc.)
  5. Detect meeting invitations and add them to Google Calendar
  6. Remind users of upcoming meetings (today's agenda)
  7. Provide an AI chatbot agent for email-related queries and actions
  8. Waitlist management with specialized landing page and admin dashboard


PROBLEM IT SOLVES
-----------------
Professionals receive 100+ emails per day. Most email clients present all
emails as equally important, causing:

  - Important tasks to be missed or delayed
  - Meeting invites to be buried and forgotten
  - Hours wasted reading low-priority emails
  - Cognitive overload from inbox noise

Decamp acts as an intelligent layer on top of Gmail, surfacing what matters
most, scheduling meetings automatically, and allowing users to chat with an
AI agent that can answer questions about their emails.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FEATURE 1 — GMAIL INTEGRATION
  ✦ Sign in with Google (OAuth 2.0), granting Gmail read access
  ✦ Fetch latest N emails from the inbox (configurable; default 30)
  ✦ section to fetch emails exactly as they appear in Gmail
  ✦ Retrieve full email body (plain text + HTML)
  ✦ Support email attachments metadata display (name + size)
  ✦ Refresh token handling for persistent sessions

FEATURE 2 — AI PRIORITIZATION (Gemini AI)
  ✦ Each email is analyzed by Gemini and assigned:
       Priority Level      : high | medium | low
       Urgency Tier        : "Immediate Attention" | "Needs Response" | "Can Wait"
       Needs Response      : boolean (does the user need to reply?)
       Deadline Detected   : date/time string or null
       Category            : work | finance | personal | meeting | form | newsletter | other
       Summary             : 2-3 sentence plain-English summary
       Action Items        : bulleted list of things the user needs to do
       Sentiment           : positive | neutral | negative | urgent
       Meeting Detected    : boolean
       Meeting Details     : {title, date, time, duration_minutes, location}
       Form Detected       : boolean (contract, survey, sign-up required)
       Waiting On          : boolean (user is waiting for a response)

FEATURE 3 — EMAIL SUMMARIZATION & GROUPING
  ✦ Dashboard displays emails grouped by category tabs:
       All | Immediate | Meetings | Forms | Waiting | Low Priority
  ✦ Each email card shows: sender, subject, AI summary (2 lines), priority badge
  ✦ Clicking an email opens a reading pane with full AI analysis
  ✦ Thread summarization: for multi-email threads, one combined summary
  ✦ Search bar: full-text search across synced emails

FEATURE 4 — GOOGLE CALENDAR INTEGRATION
  ✦ When a meeting is detected in an email, a one-click "Add to Calendar" button
    appears in the reading pane
  ✦ Before adding: check for conflicts using Google Calendar's freebusy API
  ✦ If no conflict: create the event and return a link to Google Calendar
  ✦ If conflict: show a modal listing the conflicting event and ask user to
    choose a new time
  ✦ Today's Agenda panel: right sidebar shows all calendar events for today,
    fetched live from Google Calendar
  ✦ Upcoming meetings reminder: a prominent banner shows the next meeting within
    the next 60 minutes ("You have a meeting in 15 min: Team Standup")

FEATURE 5 — MEETING REMINDERS
  ✦ On app load, check if any calendar event starts within 60 minutes
  ✦ Display a persistent reminder banner at the top of the dashboard
  ✦ In-app notifications (browser Notification API) when a meeting is 5 min away
  ✦ The chatbot can also be queried: "What meetings do I have today?"

FEATURE 6 — AI CHATBOT AGENT
  ✦ A floating chat widget (bottom-right corner) powered by Gemini AI
  ✦ The chatbot is given full context of the user's synced emails and today's
    calendar events as a system prompt
  ✦ Capabilities:
       - "Summarize my emails from John"
       - "What are my action items today?"
       - "Do I have any meetings tomorrow?"
       - "Which emails need urgent attention?"
       - "Which emails can I ignore for now?"
       - "Schedule a meeting with Dr. Asante on Friday at 2 PM"
       - "Draft a reply to [sender]" (returns AI-generated draft)
  ✦ Conversation history is maintained per session
  ✦ Chatbot responses stream in real-time (SSE)

FEATURE 7 — WAITLIST & ADMIN DASHBOARD
  ✦ High-conversion landing page with glassmorphism waitlist form
  ✦ Dedicated Admin Dashboard for data-driven management
  ✦ Features: CSV export, bulk delete, status updates, analytics charts
  ✦ Secure access via admin-specific role checks

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TABLE: users
  id                  INTEGER PRIMARY KEY
  google_id           VARCHAR(128) UNIQUE
  email               VARCHAR(256)
  name                VARCHAR(256)
  picture_url         VARCHAR(512)
  access_token        TEXT
  refresh_token       TEXT
  token_expiry        DATETIME
  created_at          DATETIME

TABLE: emails
  id                  INTEGER PRIMARY KEY
  user_id             INTEGER (FK → users.id)
  gmail_id            VARCHAR(128) UNIQUE
  thread_id           VARCHAR(128)
  sender              VARCHAR(256)
  sender_email        VARCHAR(256)
  subject             VARCHAR(512)
  snippet             TEXT
  body                TEXT            (plain text)
  body_html           TEXT            (raw HTML)
  received_at         DATETIME
  -- AI fields --
  priority            VARCHAR(20)     (high | medium | low)
  urgency_tier        VARCHAR(50)     (Immediate | Needs Response | Can Wait)
  needs_response      BOOLEAN
  summary             TEXT
  action_items        TEXT            (JSON array of strings)
  category            VARCHAR(50)
  sentiment           VARCHAR(20)
  deadline_detected   VARCHAR(100)
  meeting_detected    BOOLEAN
  meeting_title       VARCHAR(256)
  meeting_date        VARCHAR(50)
  meeting_time        VARCHAR(50)
  meeting_duration    INTEGER         (minutes)
  meeting_location    VARCHAR(256)
  form_detected       BOOLEAN
  form_description    TEXT
  waiting_on          BOOLEAN
  calendar_event_id   VARCHAR(256)    (Google Calendar event ID, if scheduled)
  synced_at           DATETIME

TABLE: waitlist_entries
  id                  INTEGER PRIMARY KEY
  email               VARCHAR(256) UNIQUE
  name                VARCHAR(256)
  role                VARCHAR(100)
  use_case            TEXT
  status              VARCHAR(50)     (pending | invited | accepted)
  created_at          DATETIME

  SECTION 6: API ENDPOINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

AUTH
  GET  /login                       → Redirect to Google OAuth
  GET  /auth/callback               → OAuth callback, save tokens, redirect to /
  GET  /logout                      → Clear session, redirect to /login

FRONTEND PAGES
  GET  /                            → Serve dashboard (index.html)
  GET  /login_page                  → Serve login page (login.html)

EMAIL API
  POST /api/emails/sync             → Trigger Gmail sync (fetches + analyzes)
  GET  /api/emails                  → List all synced emails (with filters)
       ?category=meeting            → Filter by category
       ?tier=immediate              → Filter by urgency tier
       ?search=keyword              → Full-text search
       ?limit=30&offset=0          → Pagination
  GET  /api/emails/{id}             → Get single email full details
  GET  /api/emails/stats            → Dashboard stats (counts per tier/category)

CALENDAR API
  GET  /api/calendar/today          → Get today's Google Calendar events
  GET  /api/calendar/upcoming       → Get events in next 60 minutes (reminders)
  POST /api/calendar/add            → Add event; check conflicts first
       Body: {email_id, title, date, time, duration, description}
  GET  /api/calendar/conflicts      → Check if a time slot is free
       ?date=2026-03-15&time=14:00&duration=60

CHATBOT API
  POST /api/chat                    → Send a chat message
       Body: {message, session_id}
       Returns: streaming SSE response (text/event-stream)
  GET  /api/chat/history            → Get conversation history for session
       ?session_id=abc123

WAITLIST & ADMIN API
  POST /api/waitlist                → Submit waitlist form
  GET  /api/admin/waitlist          → List all entries (Admin only)
  PATCH/DELETE /api/admin/waitlist  → Update or remove entries
  GET  /api/admin/stats             → Waitlist analytics (signups over time)

  Mitigation: Truncate body to 4,000 characters before sending to Gemini

  Mitigation: Use Render.com for deployment (HTTPS automatic),
